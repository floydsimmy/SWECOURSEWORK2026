# Peer Review

Per the coursework specification, each five-member team has 100 peer
points to distribute (5 x 20). Marks are non-negative integers and
sum to exactly 100. They reflect each member's contribution to the
project as a whole.

## Agreed marks

| Member   | Role                                  | Peer mark |
| -------- | ------------------------------------- | --------- |
| Floyd    | Technical Lead & Integrator (dev)     | 20        |
| Maysarah | Game Logic Engineer (dev)             | 20        |
| Member C | Product Owner                         | 20        |
| Member D | QA Lead                               | 20        |
| Member E | Scrum Master & Documentation Owner    | 20        |
| **Sum**  |                                       | **100**   |

The team agreed each member contributed equivalently within their
role, with no ghosts and no significant under-contribution. Evidence
supporting equal allocation:

- Each member is named in the contributions table of every sprint
  document (`docs/sprints/sprint_*.md`) and in the meeting minutes
  (`docs/meetings/`).
- Each member produced a measurable artifact every sprint:
  - Floyd: GUI screens, integration, code review.
  - Maysarah: engine modules, unit tests.
  - Member C: requirements doc, acceptance criteria, scope decisions.
  - Member D: system test report, regression runs, bug log.
  - Member E: meeting minutes, sprint start/end docs, submission pack,
    demo video.
- No member missed more than one of the eight scheduled meetings.
